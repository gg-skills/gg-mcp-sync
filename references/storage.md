# MCP Sync Runtime Storage

MCP Sync stores per-project runtime data in a single untracked directory at the target project root:

```text
.mcp-sync/
  env              # secret dotenv values
  env.example      # generated placeholders/template values
  state.json       # selected services, transports, editor scopes, sync metadata
  instructions/    # generated manual setup instructions for UI-only tools
  backups/         # project-local backups of editor config files
```

## Why `.mcp-sync/`

- Keeps secret values and mutable operator choices out of the reusable skill checkout.
- Avoids scattering tool-owned state across root dotfiles.
- Gives every host project one obvious ignore pattern.
- Makes it clear which files are generated local state versus editor-owned config files.

## Required target-project ignore rule

```gitignore
.mcp-sync/
```

Editor configuration files generated by `mcp-sync apply` are separate from `.mcp-sync/`. Those files may still be sensitive because MCP configs can contain URLs, headers, or command environment variables; review each target project’s policy before committing generated editor configs.
